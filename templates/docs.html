Face Verify — Integration API

Verify identity from a face and get a signed allow/deny. Base URL is this server. Download the machine-readable spec: openapi.yaml.

Authentication

Send your key on every request: 
X-API-Key: fk_your_key_here
Each key is scoped to your tenant (your users are isolated) and has a role:
RoleCan do
admineverything (enrol, delete, list, verify)
verifyrecognition only — verify / identify / embed / compare

Endpoints

POST /v1/enroll admin

Enrol a user from one or more images.

curl -k https://HOST/v1/enroll -H "X-API-Key: fk_x" -H "Content-Type: application/json" \
  -d '{"user_id":"alice","images":["<b64>","<b64>"]}'
POST /v1/enroll/bulk admin

Enrol many people in one call. Each entry has images or embeddings.

{"people":[{"user_id":"a","images":["<b64>"]},{"user_id":"b","embeddings":[[...]]}]}
POST /v1/verify verify

1:1 (with user_id) or 1:N. Pass image, or frames+token for a live head-turn check.

curl -k https://HOST/v1/verify -H "X-API-Key: fk_x" -H "Content-Type: application/json" \
  -d '{"user_id":"alice","image":"<b64>"}'
# -> {"success":true,"user_id":"alice","score":0.97,"signature":{...}}
POST /v1/identify verify

Who is this? Returns the matched user_id.

POST /v1/embed  ·  /v1/compare verify

Stateless: turn an image into a 512-d vector you store yourself, then compare a probe against your vectors.

{"probe":{"image":"<b64>"},"references":[{"embedding":[...]}],"threshold":0.4}
GET /v1/users  ·  POST /v1/users/delete manage · delete

List, or delete one (user_id) or many (user_ids).

POST /v1/users/export  ·  /v1/users/purge manage · delete

Data-subject access (what we hold) · erase all users in your tenant (confirm:true).

GET /v1/usage  ·  /v1/challenge  ·  /v1/health

Your monthly usage · a liveness head-turn token · readiness (no auth).

Trusting the result

Verify/compare responses include an HMAC signature keyed by your signing secret, so a tampered response is detectable. SDKs verify it for you (fv.verify_signature(r)).

Drop-in widget (no code)

Add face verification to any web app with one script tag — see the live embed + preview at /widget: 
<script src="/widget.js"></script>
<face-verify base="THIS_HOST" api-key="fk_verify_key"></face-verify>
